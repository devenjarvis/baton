### Changed

- Rewrote the planner Sonnet prompt to produce richer, agent-executable plans. The previous prompt enforced a 5-section structure (Goal / Context / Tasks / Verification / Not in scope) with a 400-word output cap and one sentence of guidance per section. Research consensus across Plan-and-Solve prompting (Wang et al, ACL 2023), TDAD (Test-Driven Agentic Development), Anthropic's "Building Effective Agents," Superpowers, and OpenSpec is that plan *completeness* — not brevity — predicts build-agent success. The new prompt mandates eight sections (Goal / Spec / Context / Reuse / Risks / Tasks / Verification / Not in scope), removes the word cap, and requires per-task labeled sub-bullets (`Files: ...`, `Signatures: ...`, `Test first: ...`, `Implement: ...`, `Verify: ...`) with file:line targeting. Test-first ordering is mandatory: every task names a failing test before its production change. Cognitive-load mitigation is baked in via density rules (Goal one sentence, Spec items one line, imperative task names, one-line labeled sub-bullets) and explicit forbidden language ("TBD", "appropriate error handling", filler phrases) so the human reviewer's scan path stays short while the building agent gets executable detail. The checkbox-counting constraint that drives `[task N]` commit prefixes is preserved: sub-bullets use `  - ` indented bullets, never `- [ ]`, since a stray checkbox anywhere in the document shifts the global task index. The revise prompt now references the eight-section structure and drops its 400-word cap as well.
